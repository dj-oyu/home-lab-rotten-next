# Proof ディレクトリ

このディレクトリは、検証・実証用のファイルを格納するためのものです。

## 目的

- 脆弱性の検証コード（Proof of Concept）の保管
- 研究成果の実証データの保存
- テストケースや実験結果の記録

## 使用ガイドライン

### ファイル配置
- 検証コードは機能や脆弱性の種類ごとにサブディレクトリを作成して整理してください
- ファイル名は内容が分かりやすい名前をつけてください

### セキュリティ上の注意
⚠️ **重要**: このディレクトリに配置するファイルには以下の注意が必要です：

1. **悪用厳禁**: すべてのコードは研究・教育目的のみで使用してください
2. **機密情報の除外**: 実際の認証情報や個人情報を含めないでください
3. **隔離環境**: 検証は必ず隔離された環境で実施してください
4. **ドキュメント化**: 各検証コードには目的と使用方法を明記してください

### ファイル構成例
```
proof/
├── rce/                    # RCE関連の検証
│   ├── poc-001/           # 特定の脆弱性のPoC
│   │   ├── exploit.js
│   │   └── README.md
│   └── poc-002/
├── xss/                    # XSS関連の検証
└── results/                # 検証結果のログやスクリーンショット
```

## ベストプラクティス

1. **ドキュメント必須**: 各PoCには必ずREADME.mdを添付し、目的・動作・結果を記録する
2. **再現可能性**: 他の研究者が再現できるよう、手順を詳細に記述する
3. **倫理的配慮**: 研究倫理を守り、責任ある開示を心がける
4. **バージョン管理**: Gitで変更履歴を管理し、重要な発見は適切にコミットする

## 注意事項

このディレクトリの内容は研究目的のものであり、実際のシステムへの攻撃に使用することは法律で禁止されています。
必ず法的および倫理的なガイドラインに従って使用してください。
